# AGENTS

## プロジェクトについて

プロジェクト名: **Mocolamma**

ターゲットプラットフォームは次の通り:

- macOS 15.0またはそれ以降
- iOS 18.0またはそれ以降
- iPadOS 18.0またはそれ以降
- visionOS 2.0またはそれ以降（iPadアプリとして）

### プロジェクト概要

Mocolammaは、オープンソースのAIソフトウェアである「Ollama」が提供するOllama APIを使って、ローカルネットワーク上に存在する複数のOllamaサーバーを管理・制御するためのマネージャーアプリケーションである。各機能の詳細については`/README.md`を参照のこと。

Mocolammaは主に`Swift`と`SwiftUI`を使って開発され、必要に応じて外部パッケージを活用して機能の実装が行われている（例: `CompactSlider`など）。

新機能を実装する際など、Ollama APIの詳細や使い方が知りたい場合は、`/docs/for-ai/Ollama-API-Document_For-AI.md`を参照のこと。このドキュメントはOllamaの公式リポジトリで提供されているAPIドキュメントの特定の時点のコピーである（オリジナルリンクはドキュメント内の上部に記載されている）。

## 開発方針と注意事項について

### 開発方針

Mocolammaの開発は、次の方針で進めていく。

#### 言語・ローカライズに関して

- デフォルトのアプリ言語は英語で、複数の言語にローカライズを行ってマルチ言語対応を行う
  - デフォルトのUIテキストやローカライズキーは英語で記載すること
- ローカライズにはString Catalogシステムを使用する（通常の`Text('text')`のようなテキストはローカライズキーとして認識される）
- ユーザーに表示されるエラーメッセージやアラートなどに表示されるエラーメッセージもきちんと翻訳されるように実装する
  - ただし、`print()`文で出力されるデバッグログやコンソール出力はローカライズせず、英語で出力されるようにする必要がある

#### 新機能の実装について

- 新機能を実装する際やリファクタリングを行う際、今後の開発持続性やメンテナンス性を高めるため、可能な限り再利用可能なコンポーネントを作成して使用すること
- あらゆるユーザーが簡単にMocolammaを使用できるよう、様々なアクセシビリティに配慮した設計・実装を行うこと
  - 例えば、レイアウトの関係により、ラベルを非表示にしたスイッチやピッカーでもVoiceOverで読み上げられるようにラベルテキストを設定すること

### 注意事項

Mocolammaの機能やOllama APIには、注意が必要な点がいくつかある。新機能を実装する際などはこれらの点に注意しつつ開発を進める必要がある。

#### サーバーリスト選択について

「サーバー」画面にはMocolammaが制御の対象とするサーバーを選択するためのリストがあるが、リスト項目の「選択」動作には次の点に注意が必要である。

- 「リストの選択」は「サーバーの選択」とは異なる
  - SwiftUIのリスト項目をクリックまたはタップしたときにリスト項目が選択され、選択された項目がハイライトされる（`selection`バインディング）が、これは「リスト項目の選択」であり、「制御対象サーバーの選択」とは異なる
  - Mocolammaでは、複数のサーバーがリストされている際、リスト項目の選択はサーバーの選択ではなく、その項目の詳細情報（そのサーバー上で動作しているOllamaのバージョンや実行中のモデル数など）を確認するためのプレビューに使用されており、サーバー選択の確定は別のアクション（コンテキストメニューの「選択」など）として用意されている（例えば、サーバーリストにサーバーAとサーバーBがあり、サーバーAが制御対象として選択された状態でリスト項目からサーバーBを選択すると、インスペクタにサーバーBの情報がプレビューとして表示される）
  - 制御対象として選択されたサーバーはその項目にチェックマークが表示されるようになっており、リスト項目の選択（`selection`バインディング）とは切り離して考える必要がある

#### モデルリストについて

Mocolammaには、Ollamaサーバー上に存在するモデルリストを様々な場所で表示することができるが、注意点が存在する。

- 「モデル」画面に表示されるモデルリストと「チャット」や「画像生成」画面に表示されるモデルピッカーは全て分離する必要がある
  - モデルリストは「モデル」画面のリストと、「チャット」および「画像生成」画面のツールバーにあるモデルピッカーの中に表示される
  - それぞれに表示されるモデルリストは適切なモデルが表示されるように開かれているため、これらのモデルリストはそれぞれ別々であり、完全に分離して考える必要がある

#### Ollama APIについて

- Ollamaサーバー上に存在するモデル情報を取得する際、Ollama APIの`/api/show`エンドポイントから返されるモデル情報（`model_info`）には、モデルの種類によって異なるキー（例: `llama.context_length`または、`mistral.context_length`など）でコンテキスト長が提供されるため、`context_length`の値を抽出する際は、キーのプレフィックスに依存せず、`.context_length`で終わるキーを検索して値を取得する必要がある
